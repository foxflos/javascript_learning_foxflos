Вопросы к этому заданию
Какое будет выведено значение: let x = 5; alert( x++ ); ?
5, так как используется постфиксная форма инкремента.

Чему равно такое выражение: [ ] + false - null + true ?
NaN - Not a Number  - Мы пытаемся произвести мат. действия не над числами.

Что выведет этот код: let y = 1; let x = y = 2; alert(x); ?
2, так как порядок порядок присваивания идет справа налево. Оператор присваивания - правоассоциативный (справа налево).

Чему равна сумма [ ] + 1 + 2?
12, так как оператор ... + ... левоассоциативный (порядок выполнения слева направо), значит сначала он обрабатывает [] + 1 как строки, а затем к полученному строковому значению присоединяет 2 в виде строки (конкатенация).

Что выведет этот код: alert( "1"[0] )?
1, так как происходит операция получения элемента массива по его индексу (0). Так как слева не массив, а строка, то берётся символ по индексу 0, и этим символом оказывается "1".

Чему равно 2 && 1 && null && 0 && undefined ?
null, так как это первое "ложное" значение в списке.

Есть ли разница между выражениями? !!( a && b ) и (a && b)?
Конструкция !! приводит операнд к логическому типу (true/false). 
Таким образов !!( a && b ) всегда возвращает true или false, а результат (a && b) определяется типом операндов.

Что выведет этот код: alert( null || 2 && 3 || 4 ); ?
3. Сначала выполняется логическое И - 2 && 3, оба значения true, значит возвращает последнее true значение - 3, а уже затем выполняется логическое ИЛИ - возвращает первое true значение.

a = [1, 2, 3]; b = [1, 2, 3]; Правда ли что a == b ?
Нет. Массивы занимают разные области в памяти, и сравнение идет по ссылкам на область в памяти, а не по конкретным значениям в самих массивах.

Что выведет этот код: alert( +"Infinity" ); ?
Infinity, так как унарный плюс + преобразует строку в число, а Infinity - это значения типа данных "число".

Верно ли сравнение: "ёжик" > "яблоко"?
Да. При сравнении сравниваются коды символов.


Чему равно 0 || "" || 2 || undefined || true || falsе ?
2, так как это первое true значение в списке.
